在 Level Set ILT 中处理 Forbidden Areas（禁止区域，例如保留区域、SRAF 禁布区等）是一个非常常见的约束问题。

既然你提到的是控制 **Phi Image 的高度（数值）** 必须在阈值之下（通常是 $\phi < 0$ 以确保不打印），最标准且利于优化的方法是引入一个 **惩罚项（Penalty Term）** 到总 Cost Function 中。

这种方法被称为 **“软约束”（Soft Constraint）**，因为它通过梯度下降将 $\phi$ “推”回允许的范围内。

以下是具体的构造方法：

### 1. 定义 Forbidden Mask (ROI Map)

首先，你需要定义一个二维矩阵（Mask），标记哪些区域是禁止的。

记为 $W_{forbid}(x, y)$：

- **$W_{forbid}(x, y) = 1$**：该像素点属于 Forbidden Area。
- **$W_{forbid}(x, y) = 0$**：该像素点是自由优化区域。

### 2. 构造 Cost Function ($J_{penalty}$)

我们需要构造一个函数，当 $\phi$ 在禁止区域内的值超过阈值 $T$（例如 $T=0$）时，产生巨大的 Cost；而当 $\phi$ 小于阈值时，Cost 为 0。

最常用的方法是 **二次单边惩罚函数 (Quadratic One-sided Penalty / Barrier Function)**。

设 $T_{limit}$ 为你允许的最大高度（通常设为 $0$ 或稍微保留一点安全余量如 $-0.1$）。

$$J_{forbid} = \frac{1}{2} \gamma \sum_{x,y} W_{forbid}(x,y) \cdot \left[ \max(0, \phi(x,y) - T_{limit}) \right]^2$$

- **$\gamma$ (Gamma)**: 惩罚权重。这是一个非常大的正数（例如 $10^3$ 或 $10^4$），用于强行压制违规区域。
- **$\max(0, \dots)$**: 类似于 ReLU 激活函数。只有当 $\phi$ 高度超过 $T_{limit}$ 时才计算 Cost。如果 $\phi$ 已经是负数（合规），则此项为 0，不影响梯度。

### 3. 构造 Gradient ($\frac{\partial J_{forbid}}{\partial \phi}$)

根据链式法则，对上述 Cost 函数求导，我们可以得到该惩罚项对 $\phi$ 的梯度。这个梯度将直接叠加到光刻误差的梯度上。

$$\frac{\partial J_{forbid}}{\partial \phi(x,y)} = \gamma \cdot W_{forbid}(x,y) \cdot \max(0, \phi(x,y) - T_{limit})$$

#### 梯度的物理意义

- **在安全区域 ($W=0$)**：梯度为 0。
- **在禁止区域 ($W=1$) 但 $\phi$ 达标 ($\phi < T_{limit}$)**：梯度为 0。
- **在禁止区域且 $\phi$ 违规 ($\phi > T_{limit}$)**：梯度为正值 $\gamma \cdot (\phi - T_{limit})$。

在梯度下降更新公式 $\phi_{new} = \phi_{old} - \alpha \cdot \nabla J$ 中，这个**正梯度的负方向**会将 $\phi$ 的数值向下拉，直到它低于 $T_{limit}$。

### 4. 进阶：基于 Sigmoid/Heaviside 的惩罚（针对二值化光罩）

如果你更关心光罩最终的透过率（Mask Transmission）必须为 0，而不是严格限制 Level Set 的数值深度，可以使用基于 Sigmoid 的惩罚。这在梯度平滑性上表现更好。

令 $S(\phi)$ 为用于近似光罩的 Sigmoid 函数：


$$
S(\phi) = \frac{1}{1 + e^{-\beta \phi}}
$$


Cost Function:


$$
J_{forbid} = \frac{1}{2} \gamma \sum_{x,y} W_{forbid}(x,y) \cdot (S(\phi(x,y)))^2
$$


(这里的逻辑是：如果该区域是 forbidden，那么 Mask Transmission $S(\phi)$ 应该尽可能接近 0)。

Gradient:


$$
\frac{\partial J_{forbid}}{\partial \phi} = \gamma \cdot W_{forbid} \cdot S(\phi) \cdot \frac{\partial S}{\partial \phi}
$$


其中 $\frac{\partial S}{\partial \phi} = \beta S(\phi) (1 - S(\phi))$。

- **优点**：梯度非常平滑，适合 L-BFGS 等高级优化器。
- **缺点**：如果 $\phi$ 变成非常大的正数，Sigmoid 导数会趋近于 0（梯度消失），导致拉不回来。通常配合正则化使用效果更好。

### 5. 替代方案：硬截断 (Hard Constraint / Clamping)

除了计算 Cost 和 Gradient，还有一种更暴力但也极其有效的方法：**在每次迭代更新后直接修改 $\phi$ Image**。

步骤如下：

1. 计算正常的 Litho Gradient $\nabla J_{litho}$。

2. 更新 $\phi_{temp} = \phi - \alpha \cdot \nabla J_{litho}$。

3. 强制截断 (Projection)：

   对于所有 $(x, y)$，如果 $W_{forbid}(x, y) = 1$：

   

   $$\phi_{new}(x,y) = \min(\phi_{temp}(x,y), T_{limit})$$

   

   否则：

   

   $$\phi_{new}(x,y) = \phi_{temp}(x,y)$$

- **优点**：绝对保证 Forbidden Area 没有任何图形生成，计算开销极小。
- **缺点**：严格来说破坏了优化器的连续性假设（相当于在流形上做投影），但在工程实践中对于 Level Set 方法通常非常稳定。

### 总结建议

如果你正在使用基于梯度的优化器（如 L-BFGS-B 或最速下降法）：

1. **首选方案**：使用 **Quadratic Penalty (方案 2)**。它线性地增加梯度反作用力，数值稳定性好，且不会像 Sigmoid 那样出现梯度消失。
2. **权重设置**：开始时 $\gamma$ 可以设小一点，随着迭代进行逐步增大（Continuation Method），以确保最终约束被严格满足。

将几何图形（Polygons）转化为用于计算 Quadratic Penalty 的栅格化数据（即 Weight Map $W_{forbid}$），在 Level Set ILT 中是一个看似简单但极易“翻车”的步骤。

如果处理不好，会导致优化器震荡、收敛变慢，或者产生“锯齿状”的光罩边缘。

以下是使用 **Quadratic Penalty** 时，将几何图形 Render 到栅格（Grid）上需要特别注意的 **5 个关键点**：

### 1. 必须使用抗锯齿（Anti-Aliasing / Soft Rasterization）

这是最重要的一点。

不要将几何图形简单地二值化为 0 和 1 的掩膜（Binary Mask）。

- **问题**：如果在网格上生成的 $W_{forbid}$ 是硬边（Hard Edge，只有0或1），那么当 $\phi$ 的等值面跨过像素边界时，惩罚项会发生跳变。这会导致梯度在像素边缘处不连续，使得基于梯度的优化器（如 L-BFGS）难以精确定位边界，容易在像素边缘反复震荡。
- **解决方案**：使用 **Area Sampling** 或 **Super-sampling** 生成灰度掩膜（Grayscale Mask）。
  - 如果一个像素只有 50% 被 Forbidden Polygon 覆盖，那么该像素的 $W_{forbid}$ 值应为 **0.5**。
  - **效果**：这创造了一个“软墙（Soft Wall）”，梯度的惩罚力度随着像素被覆盖的程度线性变化，引导优化器平滑地将图形推离边界。

### 2. 网格对齐（Grid Alignment）与采样中心

你需要明确你的 Phi Image 像素代表的是什么：

- **Pixel-as-Area（像素作为区域）**：通常推荐。
- **Pixel-as-Point（像素作为采样点）**：容易产生偏差。

注意事项：

当你的 Forbidden Area 边缘恰好落在像素中心时，如果处理不当，可能会导致半个像素的误差（CD Error）。

- **最佳实践**：确保光刻模拟引擎（Optical Model）的坐标系与你 Rasterize 的坐标系完全一致。通常建议使用 **“像素中心（Pixel Center）”** 坐标进行几何判断。

### 3. 设置“安全缓冲” (Safety Margin / Buffering)

在 ILT 中，数学上的边界和物理上的边界不同。

- **问题**：如果你严格按照 Forbidden Area 的几何边缘生成 $W_{forbid}$，Level Set 可能会紧贴着边缘生成图形。由于光刻邻近效应，这可能导致实际打印出的图形越界（Bridge 或 Short）。
- **解决方案**：在 Rasterize 之前，对 Forbidden Polygon 进行微小的 **膨胀（Dilate / Buffer）**。
  - 例如：如果设计规则要求间距 (Space) 至少 40nm，你可以将 Forbidden Area 向外扩张 5-10nm 作为安全余量，然后再转化为 $W_{forbid}$。

### 4. 冲突检测（Sanity Check）

在开始优化前，必须检查 **Target Image（目标图形）** 和 **Forbidden Area** 是否存在重叠。

- **矛盾**：如果 Target 要求某处必须是 1（透光），而 Forbidden Area 要求该处必须是 0（遮光），Quadratic Penalty 会与主目标函数（EPE Cost）打架。
- **后果**：优化器会卡在中间，生成一个高度不高不低的 $\phi$，导致光罩出现“灰度”区域，或者产生极大的梯度值破坏数值稳定性。
- **处理**：在代码中加入预处理步骤，如果发现 Target 与 Forbidden 重叠，要么报错，要么根据优先级（通常 Forbidden 优先级更高）自动修改 Target。

### 5. 权重图的梯度平滑性 (Smoothness of the Weight Map)

除了抗锯齿，有时为了更好的收敛性，会对 $W_{forbid}$ 进行轻微的 **高斯模糊（Gaussian Blur）**。

- 原理：

  回忆梯度公式： $\frac{\partial J}{\partial \phi} = \gamma \cdot W_{forbid} \cdot (\phi - T)$。

  如果 $W_{forbid}$ 即使经过了抗锯齿，边缘依然很陡峭（从 0 突变到 1 仅在 1 个像素内），这在数值上依然是一个很强的“撞墙”感。

- **做法**：对生成的 $W_{forbid}$ 做一个半径很小（比如 sigma=0.5 或 1.0 pixel）的高斯模糊。这会让惩罚力场向外扩散一点点，让 Level Set 在离墙还有几个像素远的时候就开始感受到“斥力”，从而更优雅地刹车。

### 总结：理想的流程

假设你有一个几何多边形 list `polygons`：

1. **Pre-process**: 对 `polygons` 进行 **Polygon Clipping**，确保它不与 Target 冲突，并向外做微小的 **Buffer**（例如 +5nm）。
2. **Rasterize**: 创建一个全零的 `float32` 矩阵 `W_map`。
3. **Draw**: 使用支持 **Anti-aliasing** 的库（如 OpenCV 的 `cv2.fillPoly` 加 `LINE_AA`，或者基于 `matplotlib.path` 的自定义采样）将多边形画到 `W_map` 上。
   - 边缘像素的值应当是 0.0 ~ 1.0 之间的小数。
   - 内部像素的值是 1.0。
4. **Optional Blur**: `W_map = gaussian_filter(W_map, sigma=0.8)`。
5. **Optimization**: 在计算 Cost 时使用这个平滑的 `W_map`。
